# Stage 1 — Teleoperation (live capture)

Two cooperating processes managed by `TeleopManager` (`real/teleop/manager.py`) write
to a per-episode directory `<task>/episode_<N>/`. Cadence is **30 Hz** (`FREQ = 30`,
`DELAY = 1/30`). All sizes below are for the **G1** robot
(`real/teleop/constants.py::G1_sizes`).

## 1A — `RobotDataWorker` (state + frames)

`real/teleop/worker.py`. Runs at 30 Hz. For every camera frame it writes
- one image `color/frame_xxxxxx.jpg`
- one depth file `depth/frame_xxxxxx.npy.lzma`
- one line to `robot_data.jsonl`

### `robot_data.jsonl`

One JSON object per line, one line per camera frame.
Built by `RobotDataWorker.get_robot_data` (`worker.py:429`).

**General**
- `time` (float),
- `robot_type` (str),
- `actions` (null),
- `image` (str),
- `depth` (str),

**Joint states**
- `states.arm_state` (float[14]),
- `states.leg_state` (float[15]),
- `states.hand_state` (float[14]),
- `states.hand_pressure_state` (dict),

**IMU** — the torso's inertial sensor: which way the body is oriented, plus its raw rotation and acceleration.
- `states.imu.quaternion` (float[4]), orientation encoded in quaternion
- `states.imu.rpy` (float[3]), orientation encoded with roll, pitch, yaw
- `states.imu.gyroscope` (float[3]), angular velocity (how fast it's rotating)
- `states.imu.accelerometer` (float[3]), linear acceleration along each axis

**Odometry** — how far and how fast the robot has moved through the world (estimated base pose + velocity).
- `states.odometry.position` (float[3]),
- `states.odometry.velocity` (float[3]),
- `states.odometry.rpy` (float[3]),
- `states.odometry.quat` (float[4]),

## 1B — `RobotTaskmaster` (actions / IK)

`real/teleop/master_whole_body.py`. Runs the whole-body IK loop. Every IK tick it
calls `IKDataWriter.write_data` (`writers.py:70`), appending one JSON line to
`ik_data.jsonl`.

### `ik_data.jsonl`

Time
- `armtime` (float), timestamp used to align with `robot_data` `time`
- `iktime` (float), IK solver wall-clock

**VR targets** — operator inputs the IK is tracking.
- `head_rmat` (float[3][3]), head rotation matrix from VR headset
- `left_pose` (float[6 or 7]), left wrist target pose
- `right_pose` (float[6 or 7]), right wrist target pose

**Hands**
- `right_angles` (float[7]), right-hand joint command
- `left_angles` (float[7]), left-hand joint command

**AMO commands** — commanded torso/base setpoints.
- `torso_height` (float), commanded torso height
- `torso_rpy` (float[3]), commanded torso roll/pitch/yaw
- `torso_vx` (float), commanded base x-velocity
- `torso_vy` (float), commanded base y-velocity
- `torso_vyaw` (float), commanded base yaw rate
- `torso_dyaw` (float), yaw-delta term
- `target_yaw` (float), absolute yaw target

**IK solution**
- `sol_q` (float[29]), full body solution: `[0:15]` legs+waist, `[15:29]` arms
- `tau_ff` (float[29]), feed-forward torque

## Dir layout at end of Stage 1

```
<task>/episode_<N>/
├── color/                frame_000000.jpg, frame_000001.jpg, …
├── depth/                frame_000000.npy.lzma, frame_000001.npy.lzma, …
├── robot_data.jsonl      (state, one row per camera frame, ~30 Hz)
├── ik_data.jsonl         (actions, one row per IK tick)
└── failed                (touch-file written iff user pressed `d`)
```
